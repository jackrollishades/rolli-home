System Overview

The Rolli Home quoting engine is a multi-vendor, multi-line-item pricing and order-submission system for window shade products. It reads raw vendor pricing from structured JSON databases, applies dealer discount factors, computes sell prices at a target margin, generates customer-facing quote documents, and submits orders to the PIC Business Systems portal.

Core Data Model

All components communicate through a shared set of Python dataclasses defined in models.py. The flow is: LineItemSpec (raw input) → QuotedLineItem (priced + PIC-mapped) → Quote (full document).

Key Architecture Decisions

Document format: DOCX (Word)

Decision: Generate quote documents as .docx files using python-docx.
Rationale: DOCX is editable (you can tweak a quote before sending), opens natively in Word / Google Docs, prints well, and is the industry standard for B2B quotes. PDF is appropriate for the final version — generate it from the DOCX via Word's print-to-PDF or LibreOffice. The docx skill provides battle-tested patterns for creating professional Word documents.

Quote storage: JSON files (no database)

Decision: Save each Quote as a JSON file in quotes/ using the quote number as filename.
Rationale: Volume is low (dozens of quotes per month, not thousands). JSON files are human-readable, diffable in git, require zero infrastructure, and can be trivially migrated to a database later. Quote numbers use format Q-YYYY-NNNN (auto-incremented from the highest existing file).

PIC automation: Claude in Chrome skill

Decision: Implement PIC portal automation as a Claude in Chrome skill that reads a Quote JSON and fills the PIC form, with a Playwright fallback script.
Rationale: The PIC portal uses a complex multi-step form with conditional fields. Browser automation is the most robust approach — it doesn't require reverse-engineering the API. Claude in Chrome can handle conditional field logic dynamically. A Playwright script is included as a headless fallback for batch submissions.

Tube / motor auto-selection: deterministic rule engine

Decision: Implement tube_motor_selector.py as a pure rule-based function using the practical thresholds from shade_selection_rules.md. Always expose the auto-selected values to the user before submission.
Rationale: The Rollease Acmeda eBOM calculator uses deflection math. The practical thresholds in shade_selection_rules.md cover 95%+ of standard residential installs. Auto-selection saves time but must be reviewable and overridable — never silently commit a motor selection.

PIC field control_type: derived from motor_brand

Watch-out: The PIC ControlType field maps as follows: Automate/Rollease → "ROLLEASE" (they share the same PIC motor catalog), Somfy → "SOMFY", Gaposa → handled separately (Gaposa controls appear in the Gaposa section). Manual → "CHAINWINDER CLUTCH" (standard) or crank options.
Action: pic_mapper.py must implement this derivation rule explicitly — it's not obvious from the schema alone.

WSC orders: not in PIC (Fabritec portal — Phase 1C)

Note: WSC/Fabritec orders go through Fabritec's own portal, not PIC. Phase 1B's PIC mapper only applies to SBM line items. WSC line items in a quote will appear in the quote document but require manual Fabritec portal entry until Phase 1C maps the Fabritec portal fields.

Width / Height fraction handling

Issue: PIC uses an integer + fraction dropdown for width and height (e.g., 48 + "1/4" = 48.25"). The quote engine stores dimensions as floats.
Solution: pic_mapper.py includes a float_to_pic_fraction(value) helper that splits a float into its integer part and nearest 1/8" fraction, then returns the matching PIC dropdown value. E.g., 48.25 → (48, "1/4"), 48.375 → (48, "3/8").

Phase 2+ — Future Extensions

Phase 1C — WSC / Fabritec portal automation

Map WSC line items to Fabritec's own order portal fields (field mapping TBD — requires a live session to capture the Fabritec portal schema, as was done for PIC). Same browser automation approach as PIC.

Phase 2 — Shopify customer configurator

A customer-facing Shopify product page where users select fabric, dimensions, and options. On checkout, Shopify sends an order webhook to a lightweight API endpoint (shopify_bridge.py) that converts the order into a QuoteRequest and feeds it through the quote engine. PIC submission happens automatically or on staff review. The internal quoting UI (Phase 1C) becomes the review dashboard.

Internal quoting UI (Phase 1C)

A simple Flask or FastAPI web app (runs locally, not public) with a multi-line shade configurator form. Replaces editing Python dicts or JSON by hand. Outputs a Quote, generates the DOCX, and has a "Submit to PIC" button that triggers the browser automation.
